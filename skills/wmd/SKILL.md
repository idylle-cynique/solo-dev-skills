直前のレスポンスをMarkdownファイルに書き出す。

## 手順

1. **ファイルパスを決定する**
   - 引数 `$0` が指定されていればそれをファイルパスとして使う
   - 指定がなければ、レスポンス内容から適切なファイル名を推測して提案する
   - 保存先ディレクトリが存在しない場合は作成する

2. **直前のレスポンス内容を確定する**
   - 「直前のレスポンス」とは、このスキルが呼び出される直前にアシスタントが返したメッセージ全体を指す
   - コードブロック・テーブル・リスト・ヘッダーなど、元のMarkdown構造をそのまま保持する
   - 会話上の前置き（「以下のとおりです」等）は含めず、実質的なコンテンツのみを書き出す

3. **ファイルに書き出す**
   - Write ツールでファイルを作成する
   - ファイル先頭に `# <タイトル>` がない場合、内容から適切なタイトルを1行追加する
   - 末尾の空行は1行に統一する

4. **完了を報告する**
   - 書き出したファイルパスを1行で報告する
   - 内容の再掲はしない

## ファイル名の推測ルール

引数が指定されない場合、以下の優先順位でファイル名を決める：

1. レスポンスの主題を表すスネークケース英語 or 日本語スラグ（例: `api_design_memo.md`）
2. 現在の作業ディレクトリ内の `docs/` または `.vscode/docs/` が存在すればそこに配置
3. どちらもなければ現在のワーキングディレクトリ直下に配置

## 入出力例

**入力（ユーザー発言）:**
```
markdownに書き出してください
```

**出力（アシスタント動作）:**
- 直前レスポンスの内容を `docs/api_memo.md` に書き出す
- 「`docs/api_memo.md` に書き出しました。」と1行報告

---

**入力（ユーザー発言）:**
```
/save-to-md .vscode/docs/plans/review_result.md
```

**出力（アシスタント動作）:**
- 直前レスポンスを `.vscode/docs/plans/review_result.md` に書き出す
- 「`.vscode/docs/plans/review_result.md` に書き出しました。」と1行報告
