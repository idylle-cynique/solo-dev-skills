以下の手順でGitHub Issue用のmdファイルを作成してください。

## 入力

タスクの説明: $ARGUMENTS

## 1. テンプレートの選定

まず Glob ツールで `.github/ISSUE_TEMPLATE/SOLODEV_*.md` を検索してください。

- ファイルが1件以上見つかった場合：そのファイル一覧を使用してください。
- 1件も見つからなかった場合：代わりに Glob ツールで `templates/SOLODEV_*.md` を検索してスナップショットテンプレートを使用してください。

確認したファイル一覧をもとに、タスクの説明に含まれるキーワードから最も適切なテンプレートを1つ選択してください。
下表は参考です。テンプレートが追加・削除されている場合は、実際に存在するファイルの名前とその内容から判断してください。

| テンプレート（参考）     | 使用するキーワードの目安                                                     |
| ------------------------ | ---------------------------------------------------------------------------- |
| `SOLODEV_bug-fix.md`     | バグ、不具合、エラー、修正、fix、bug、動かない、壊れている                   |
| `SOLODEV_new-feature.md` | 機能追加、新規実装、新機能、feat、feature、追加、開発、導入                  |
| `SOLODEV_refactoring.md` | リファクタリング、設計改善、コード整理、refactor、構造改善、分離、責務       |
| `SOLODEV_infra-ci.md`    | CI、CD、インフラ、GitHub Actions、ワークフロー、デプロイ、pipeline、環境構築 |
| `SOLODEV_chore.md`       | ドキュメント、依存関係、更新、タイポ、整理、不要コード削除、chore            |

判断が難しい場合は、タスクの主目的（何を達成したいか）で判断してください。

## 2. テンプレートの読み込み

ステップ1で選定したテンプレートファイルを読み込んでください。
（`.github/ISSUE_TEMPLATE/` またはフォールバックの `templates/` のいずれか、見つかった方のパスを使用）

## 3. mdファイルの生成

以下のルールに従ってissue用mdファイルを作成してください。

### ファイル出力先

`.vscode/claude/issues/` ディレクトリに保存してください。

### ファイル名

`issue-draft-<タスクを端的に表すkebab-case>.md`

### 内容の生成ルール

- テンプレートのフロントマター（`---` で囲まれた部分）は**含めない**
- テンプレートの各セクション見出しはそのまま使用する
- HTMLコメント（`<!-- ... -->`）はそのまま残す
- タスクの説明から読み取れる情報を各セクションに記入する
- 情報が不足しているセクションは空欄のまま（コメントだけ残す）にする
- タイトル候補を冒頭に追記する（例: `**タイトル候補**: feat: ○○機能の追加`）
- 絵文字は使用しない
- Issue はチケット単体で完結して読めるようにする。ローカルにしかない資料（ローカルファイルパス、`.claude/`・`.vscode/claude/` 配下の設計ノート・仕様書等）への参照や「詳細は◯◯参照」といった誘導は本文のどこにも入れず、必要な情報は本文に直接書き切る。`## 参考資料` に記載してよいのはURLまたはGitHub Issue/PR/コミットへのリンクのみとする

### タイトル候補のprefixルール

| テンプレート | prefix       |
| ------------ | ------------ |
| bug-fix      | `bug: `      |
| new-feature  | `feat: `     |
| refactoring  | `refactor: ` |
| infra-ci     | `ci: `       |
| chore        | `chore: `    |

## 参考

生成物の品質・フォーマットの参考として `examples/sample.md`（new-feature テンプレートの記入例）を参照してください。
概要・背景・タスクの粒度・完了条件の書き方を踏襲してください。

## 4. 完了報告と確認

以下を報告してください:

- 選定したテンプレートとその理由
- 生成したファイルのパス

報告したら、次を伝えて**処理を一時停止してください**。

```
下書きを <ファイルパス> に書き出しました。内容を確認してください。
問題なければ「OK」、修正があればフィードバックをお知らせください。
```

ユーザーから承認（「OK」「問題ない」「作成して」など肯定的な返答）を得られた場合のみ、次のステップに進んでください。修正フィードバックがあった場合はファイルを更新してから再度確認を求めてください。

## 5. Issueを作成する

承認を得たら、以下のコマンドで Issue を作成してください。body にはタイトル候補行とその前後の空行・先頭コメントを除いた本文（各セクション見出し以降）を渡します。`<タイトル候補>` はファイル冒頭のタイトル候補、`<ラベル>` はテンプレートに対応するラベル（new-feature → enhancement 等。任意）を使用します。

```bash
gh issue create \
  --title "<タイトル候補>" \
  --body "$(tail -n +3 <生成したファイルのパス>)" \
  --label "<ラベル（任意）>"
```

作成後、作成された Issue の URL を報告してください。
