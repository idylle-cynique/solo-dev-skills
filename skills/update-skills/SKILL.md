既存スキルの内容を読み込み、指示に従って更新してください。

## 入力

スキル名と変更内容: $ARGUMENTS

## 手順

### Step 1: 対象スキルの特定

引数からスキル名を解析し、以下の優先順位でファイルを探す：

1. `skills/<スキル名>/SKILL.md`（プロジェクト用・正規ソース）
2. `~/.claude/skills/<スキル名>/SKILL.md`（個人用）

スキル名が引数に含まれていない場合はユーザーに確認すること。

### Step 2: 既存ファイルの読み込み

以下を読み込む：

- `SKILL.md` 本体
- `examples/sample.md`（存在する場合）
- `resources/` 以下のファイル（存在する場合）

読み込んだ内容から以下を把握する：

- 現在のフロントマター（name, description, フラグ類）
- スキルの目的と構造
- 既存の手順・ルール・出力フォーマット

### Step 3: 変更内容の確認

編集に入る前に変更の意図を明確にする：

- `$ARGUMENTS` に変更内容が含まれている場合はそれに従う
- 含まれていない場合は「何を変更しますか？」とユーザーに確認する

**編集の原則：**

- 既存のフロントマター（name, description 等）は、明示的な変更指示がない限り保持する
- 既存の意図・構造を壊さない
- 変更は最小限に留め、必要な箇所だけ編集する
- 特定の例に対して過適合したルールを追加しない

### Step 4: 編集の実行

Edit ツールで対象ファイルを編集する。編集後に以下を確認する：

- [ ] description は「いつ使うか」と「何をするか」を両方含んでいるか
- [ ] description は積極的（pushy）に書かれているか
- [ ] 指示は命令形で書かれているか
- [ ] 500 行以内に収まっているか（超える場合は `resources/references/` に切り出す）
- [ ] `examples/sample.md` の内容と矛盾していないか（ある場合）

### Step 5: 差分の報告

編集完了後、以下を報告する：

- **変更したファイル**のパス
- **変更内容のサマリー**（何を・なぜ変えたか）
- **インストール済みコピーの更新要否**（後述）

## インストール済みコピーの扱い

`skills/<name>/` が正規ソースであり、以下はインストール済みコピー：

| パス | 用途 |
|------|------|
| `.claude/skills/<name>/` | `/スキル名` コマンドで呼び出される実体 |
| `.agents/skills/<name>/` | サブエージェント用コピー |

正規ソースを更新した後、ユーザーに確認する：

> 「インストール済みコピー（`.claude/skills/` / `.agents/skills/`）も同様に更新しますか？」

承諾を得た場合は同じ編集を各コピーにも適用する。
