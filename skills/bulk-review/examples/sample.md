## サマリー

認証フローとエクスポート処理の基本実装は適切です。ただし、外部 API から取得した文字列をスプレッドシートへ直接書き込む際にサニタイズが行われておらず、数式インジェクション（Formula Injection）の余地があります。マージ前に High 1 件の対応を推奨します。

## 指摘事項

### [High] スプレッドシート書き込み時に数式インジェクションの余地がある

- **該当箇所**: `src/lib/sheetWriter.js:87`
- **内容**: 外部 API から取得した文字列をそのまま `values` に渡して `spreadsheets.values.append` している。`=`, `+`, `-`, `@` で始まる値はスプレッドシート側で数式として解釈され、外部リクエスト誘発などに悪用されうる。
- **推奨対応**: 書き込み前に各セル値が数式接頭辞で始まる場合は先頭にシングルクォート（`'`）を付与してエスケープする。`valueInputOption` を `RAW` に統一する選択肢も検討する。
- **根拠**: 書き込むデータは外部 API 由来であり、内容を完全には制御できないため。

### [Medium] OAuth スコープが必要範囲より広い可能性

- **該当箇所**: `manifest.json:18`
- **内容**: `oauth2.scopes` に Drive 全体への読み書きスコープが指定されているが、実処理は特定フォルダへのシート追記のみ。
- **推奨対応**: `drive.file` など最小権限スコープへの絞り込みを検討する。

## 補足・対応不要

- [Low] `background.js:54` の `console.log` に書き込み先のシート ID が出力されている。機密ではないが、配布版ではログレベルを下げると望ましい。
- [Info] `chrome.alarms` による keepalive は意図的な実装であり、権限 `alarms` の付与は妥当（指摘ではない）。
